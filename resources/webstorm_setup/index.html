Import Settings

  1. Run WebStorm
  2. From the intro screen choose "Configure" and then "Import Settings" (note, if you don't see the intro screen like in the images below, but you do see a project screen, choose "Close Project" from the "File" menu):
  3. Select the webstorm_settings.jar file for importing and allow WebStorm to restart

Configure Deployment Server

  1. Go to the "Preferences..." menu item (under WebStorm menu on Mac, "Settings" under File menu on Windows)
  2. Select "Deployment" and you will see the comp105 server listed

Set up the Connection

  • Enter your username and password. If you are on a public computer, don't select "Save Password".
  1. Click the "Autodetect" button, which should put your site directory into the "Root path" field.
  2. Add "/public_html" after your site directory.
  3. Add your username at the end of the "Web server root URL".

Set up the Mappings

  1. Select the "Mappings" tab
  2. Click "Use this server as default"
  3. Select your project directory as the "Local path". (Clicking on the "..." button to the right of the Local Path field should get you the right folder.)
  4. Put "/" as the "Deployment path on server..." and "Web path on server..."

Confirm Server Configuration

  • Go to the menu item Tools Deployment Browse Remote Host
  • You should see a panel on the right open up and indicate that it is connecting to the server.
  • If your public_html directory shows up in that panel, you are done.
  • If there are problems, WebStorm will indicate the nature of the problem and you should read the instructions it provides on how to correct the issue.
