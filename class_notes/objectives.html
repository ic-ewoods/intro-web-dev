Learning Objectives

Introduction

Introductions

Lecture: Introductions

Students will learn

  • The course is a general survey of the skills, tools, and practices used in building websites, with the goal of preparing students for participating in real-world web development projects and building a foundation for more advanced and dynamic website development courses.
  • The instructor has experience building web pages
  • Web pages are the combination of content, design, and code

A First Page

Lecture: A First Page

Students will learn

  • Web pages use HTML, CSS, and JavaScript to present content
  • Web pages are on servers
  • FTP gets files to servers
  • Lecture notes pages provide resources they need for the day/assignment
  • Download packets help them organize their work

HTML Fundamentals

Structure

Lecture: Structure

Students will learn

  • HTML is made up of elements that are defined by tags
  • There is a minimum set of HTML elements that are required for a page to be valid HTML
  • h1..h6 and p are used to structure content
  • HTML for any site can be viewed in a web browser using View Source
  • They can build a web page from a blank page.
  • They can upload files and preview them using WebStorm
  • They can use WebStorm to check for errors
  • They can use View Page Source in Firefox to validate their document

Reinforce

  • Lecture notes pages provide resources they need for the day/assignment
  • Download packets help them organize their work

Text

Lecture: Text

Students will learn

  • Whitespace is collapsed by HTML
  • Comments don't appear in the browser
  • br is for creating a line break in a paragraph
  • Some html is for adding meaning, some is for presentation
  • More tags...
    • br
    • b / strong
    • i / em
    • sup, sub
    • abbr
    • blockquote
    • hr
    • other text markup
  • Some html they will see in the wild or even in references is deprecated
  • The HTML standard is dynamic and we are using html5

Reinforce

  • Download packets help them organize their work
  • Students can build a validating HTML document from a blank file.

Lists

Lecture: Lists

Students will learn

  • HTML has specific handlers for managing lists and these use multiple elements
  • Ordered lists are for sequential information and by default are displayed as a numbered list
  • Unordered lists are for general enumeration and by default are displayed as a bullet list
  • Description lists are for associating terms with descriptions
  • Lists can be nested to create a hierarchy of enumeration

Reinforce

  • WebStorm can validate HTML during development (helpful for nested lists!)
  • HTML can be validated using Firefox View Page Source
  • Always test the link you submit before you submit it
